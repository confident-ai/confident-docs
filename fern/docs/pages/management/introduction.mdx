## Overview

`ConfidentAI` is a programmatic admin client for your Confident AI account. It lets you manage your organization, projects, team members, roles, and API keys directly in code — the same actions you'd otherwise perform by clicking through the platform UI.

Use it whenever you want to script account management instead of doing it by hand — for example, onboarding new teammates, provisioning projects in CI/CD, keeping roles in sync with your identity provider, or rotating API keys.

## Setup

Install the SDK and configure your `ConfidentAI` client with an Organization API Key.

<Steps>

<Step title="Install the SDK">

<Tabs>

<Tab title="Python" language="python">

```bash
pip install confidentai
```

</Tab>

<Tab title="TypeScript" language="typescript">

```bash
npm install confidentai
```

</Tab>

</Tabs>

</Step>

<Step title="Set your Organization API Key">

The client reads the `CONFIDENT_ORG_API_KEY` environment variable by default. This is separate from the `CONFIDENT_API_KEY` used for tracing and evaluations, so you can configure both at once.

```bash
export CONFIDENT_ORG_API_KEY="confident_us_org_8Kj2mNpQ4rVz..."
```

</Step>

<Step title="Create a client">

<Tabs>

<Tab title="Python" language="python">

```python
from confidentai import ConfidentAI

client = ConfidentAI()
```

</Tab>

<Tab title="TypeScript" language="typescript">

```typescript
import { ConfidentAI } from "confidentai";

const client = new ConfidentAI();
```

</Tab>

</Tabs>

<Tip>

You can also pass the key explicitly instead of relying on the environment variable. This is useful when managing multiple organizations from the same process.

<Tabs>

<Tab title="Python" language="python">

```python
client = ConfidentAI(api_key="confident_us_org_8Kj2mNpQ4rVz...")
```

</Tab>

<Tab title="TypeScript" language="typescript">

```typescript
const client = new ConfidentAI({ apiKey: "confident_us_org_8Kj2mNpQ4rVz..." });
```

</Tab>

</Tabs>

</Tip>

</Step>

</Steps>

## Organization- vs Project-Scoped Operations

Most resources exist at two levels, and the client gives you a separate entry point for each:

- **Organization-scoped** resources operate across your entire organization. Access them from `client.organization()`.
- **Project-scoped** resources operate within a single project. Access them from `client.project(project_id)`.

From either entry point, you chain the resource and the action — for example, listing members:

<Tabs>

<Tab title="Python" language="python">

```python
# Across the whole organization
client.organization().members.list()

# Within a single project
client.project(project_id).members.list()
```

</Tab>

<Tab title="TypeScript" language="typescript">

```typescript
// Across the whole organization
client.organization().members.list();

// Within a single project
client.project(projectId).members.list();
```

</Tab>

</Tabs>

Both entry points require an Organization API Key.

## Manage Your Organization

You can retrieve the organization tied to your API key, or rename it.

<Tabs>

<Tab title="Python" language="python">

```python
from confidentai import ConfidentAI

client = ConfidentAI()

org = client.organization()

organization = org.get()
print(organization.id, organization.name)

# Rename the organization
organization = org.update(name="Acme Inc.")
```

</Tab>

<Tab title="TypeScript" language="typescript">

```typescript
import { ConfidentAI } from "confidentai";

const client = new ConfidentAI();

const org = client.organization();

const organization = await org.get();
console.log(organization.id, organization.name);

// Rename the organization
const renamed = await org.update({ name: "Acme Inc." });
```

</Tab>

</Tabs>

<Tip>

`client.whoami()` is a shortcut that returns the organization your API key is scoped to — handy for confirming which account a key belongs to.

</Tip>
